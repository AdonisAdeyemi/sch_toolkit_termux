My recommendation on table refresh

I would not manually append rows with JavaScript.

Instead, create a small reusable function:

reloadStudentTable()

Every action calls it:

New student

Existing student

Edit

Remove

Promote


This has several advantages:

one rendering path

fewer UI bugs

no duplicated HTML in JavaScript

much easier to maintain as the table evolves


It's a slight increase in server work but a big win for code simplicity and consistency.

Why I like this architecture

Your responsibilities become very clear:

Owns:

student information

passport

Owns:

The two modules work together, but each has a single, well-defined responsibility. That separation will make future features—like bulk enrollment, alumni, or inactive students—much easier to add without complicating either controller.
